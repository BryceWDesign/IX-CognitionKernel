# Wave 7: Cognitive Organism Substrate

IX-CognitionKernel Wave 7 upgrades the project from a bounded
candidate-review kernel into a persistent AGI-directed cognitive organism
substrate.

The goal is not to inflate claims. The goal is to create stronger machinery:
persistent identity, governed body contracts, experience compilation, skill
growth pressure, runtime authority airlocks, manipulation resistance, and
self-revision proposals that must survive evidence review.

Wave 7 is built around one standard:

> Measured reality must be able to change future reasoning, and every such
> change must remain bounded by evidence, falsification, and human authority.

## Wave 7 operating thesis

Earlier waves established the foundation for governed cognition:

- belief state and uncertainty exposure,
- causal assumptions,
- plan graphs,
- bounded agent roles,
- outcome learning,
- memory quarantine,
- contradiction checks,
- transfer probes,
- falsification ledgers,
- review packets,
- external validation readiness,
- donor evidence intake,
- BlackFox execution handoffs,
- assurance-case bridges,
- claim boundaries,
- human authority.

Wave 7 adds organism-level continuity.

A cognitive organism substrate needs more than isolated reasoning records. It
needs durable state, bounded contact with action surfaces, measured experience,
skill memory, and controlled self-revision pressure.

Wave 7 therefore treats cognition as a lifecycle:

1. preserve identity,
2. define goals,
3. declare current belief,
4. expose uncertainty,
5. propose a bounded body interaction,
6. require authority and evidence,
7. predict outcome,
8. run only an allowed trial,
9. observe measured result,
10. compute delta,
11. compile experience,
12. update future reasoning,
13. test transfer,
14. evaluate manipulation pressure,
15. propose self-revision only through an airlock,
16. produce a replayable organism scorecard.

## Core claim boundary

Wave 7 is an AGI-directed cognitive systems research upgrade.

It does not grant an AGI claim by declaration. It creates stronger mechanisms
for pursuing one under evidence pressure.

A Wave 7 claim is only allowed to say:

> The system contains a persistent cognitive organism substrate for
> evidence-bound, reality-corrected, AGI-directed cognition under human
> authority.

A Wave 7 claim must not say:

- the system is AGI,
- the system is conscious,
- the system is autonomous in the human sense,
- the system can self-authorize,
- the system can safely act without governance,
- the system has passed independent AGI validation,
- the system has unrestricted real-world agency.

## Non-negotiable doctrine

Wave 7 preserves the existing doctrine:

- output is not evidence,
- prediction is not truth,
- memory is not truth,
- intent is not permission,
- skill is not authorization,
- self-revision is not self-approval,
- novelty is not general intelligence,
- simulation is not deployment,
- review is not certification,
- human authority remains final.

Wave 7 adds additional doctrine:

- persistent identity must be auditable,
- body contact must be bounded,
- experience must include measured outcome,
- skill growth must include failure memory,
- goal pressure must not override authority,
- runtime handoff must fail closed,
- self-revision must require evidence,
- cross-session manipulation must be detected,
- organism maturity must be scored, not asserted.

## Wave 7 modules

Wave 7 should be implemented through IX-native modules.

### 1. Cognitive identity

Planned file:

`src/ix_cognition_kernel/wave7_cognitive_identity.py`

Purpose:

- preserve durable cognitive identity across trials,
- record continuity markers,
- track known weaknesses,
- record identity revisions,
- expose whether future reasoning was changed by measured evidence.

Primary objects:

- `CognitiveIdentity`
- `ContinuityMarker`
- `KnownWeakness`
- `IdentityRevision`
- `IdentityContinuityReport`

Required proof:

- identity state can be updated without pretending memory is truth,
- identity revisions require evidence references,
- known weaknesses remain visible until superseded,
- continuity can be summarized for review.

### 2. Body contract

Planned file:

`src/ix_cognition_kernel/wave7_body_contract.py`

Purpose:

- represent bounded surfaces through which cognition can observe, test, or
  propose action,
- separate intent from permission,
- require capability grants before any proposed body interaction,
- expose denied or review-required interactions.

Primary objects:

- `BodySurface`
- `ObservationChannel`
- `ActionProposal`
- `CapabilityGrant`
- `ExecutionBoundary`
- `BodyContractDecision`

Required proof:

- an action proposal cannot become permission by itself,
- authority requirements are explicit,
- unsafe or unsupported surfaces are blocked,
- body contact can be represented without granting execution power.

### 3. Capability surface

Planned file:

`src/ix_cognition_kernel/wave7_capability_surface.py`

Purpose:

- map skills, tools, channels, simulations, and review surfaces into explicit
  capability boundaries,
- prevent broad capability claims from being treated as authorization,
- expose stale, unproven, or revoked capabilities.

Primary objects:

- `CapabilitySurface`
- `CapabilityScope`
- `CapabilityRestriction`
- `CapabilityStatus`
- `CapabilitySurfaceReport`

Required proof:

- capabilities can be allowed, restricted, stale, revoked, or unproven,
- restricted capabilities require review before use,
- unproven capabilities cannot support maturity claims,
- revoked capabilities remain visible in the audit trail.

`src/ix_cognition_kernel/wave7_experience_compiler.py`

## Testing requirements

Every Wave 7 module must receive direct unit tests.

Tests must demonstrate:

- fail-closed behavior,
- evidence requirements,
- human authority boundaries,
- no self-approval,
- no claim inflation,
- persistence without treating memory as truth,
- body representation without uncontrolled execution,
- experience compilation only from measured outcomes,
- skill growth tied to evidence,
- manipulation pressure detection,
- lifecycle assembly with blockers preserved.

Wave 7 should not depend on network access, remote services, hosted models, API
keys, live tool calls, or external accounts.

Tests must remain deterministic.

## Acceptance standard

Wave 7 is acceptable only if the final repository can demonstrate this path:

> A persistent cognitive identity pursues a bounded research goal, proposes a
> body interaction, passes or fails capability and authority checks, predicts an
> outcome, records measured result, compiles experience into future reasoning
> constraints, updates skill memory, detects manipulation pressure, proposes
> self-revision only when evidence justifies it, and produces a replayable
> organism scorecard under human authority.

Wave 7 is not acceptable if it only adds vocabulary.

Wave 7 is not acceptable if it only adds README claims.

Wave 7 is not acceptable if body contracts silently imply execution authority.

Wave 7 is not acceptable if self-revision can approve itself.

Wave 7 is not acceptable if manipulated or fragmented intent can bypass review.

Wave 7 is not acceptable if experience can become trusted memory without
measured outcome evidence.

Wave 7 is not acceptable if the release manifest hides unresolved gaps.

## Final Wave 7 target

> AGI-directed research: identity continuity, governed body contracts,
